Use template .github/templates/readme-template.md as base for standard structure.
Required structure: Introduction, Table of Contents, Installation, Quick Start, Usage Examples, API Reference, Build and Tests, Contributing, Dependencies, References.
Introduction: problem solved, context, technical approach; key features with checkmarks.
Table of Contents: links to all main sections.
Installation: .NET CLI commands and PackageReference.
Quick Start: minimal example in 3–5 lines.
Usage Examples: typical and advanced cases with full code.
API Reference: key signatures, parameters and returns.
Build and Tests: dotnet build/test/format with filters.
Contributing: Git flow, guidelines, semantic commits.
Dependencies: separate runtime and development.
References: technical links, issues, changelog.
Format: sections with separators (---); fenced code blocks; checkmarks for features; keep concise and practical.
